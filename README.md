## About The Project

**HarmonAI** is an AI-powered platform designed for musicians and enthusiasts who want to learn how to play their favorite songs but lack the chords. 

By simply uploading an MP3 file, HarmonAI leverages advanced machine learning models to transcribe the chords for you. Additionally, users can explore a comprehensive database of previously transcribed songs. The system also includes a robust administrator dashboard for managing datasets, retraining the model, and monitoring AI performance metrics.

## Key Features

### User Features
* **AI Chord Transcription**: Upload MP3 files and receive accurate chord transcriptions generated by our custom TensorFlow models.
* **Song Database**: Search and discover chords for songs that have already been transcribed by the community.
* **User Accounts**: Create an account to save your favorite transcriptions and keep track of your library.

### Administrator Features
* **Model Management**: Upload new training data and manually trigger the training of a new machine learning model.
* **Version Control**: Seamlessly switch between different versions of the trained AI models.
* **Performance Analytics**: View detailed model performance statistics and compare different models side-by-side to ensure optimal accuracy.

## Tech Stack

* **Frontend**: [Vue.js 3](https://vuejs.org/) powered by [Vite](https://vitejs.dev/), using Pinia for state management and Chart.js for admin analytics.
* **Backend**: [Django](https://www.djangoproject.com/) REST API handling authentication, user management, and serving predictions.
* **Database**: [MongoDB](https://www.mongodb.com/) for scalable data storage.
* **Machine Learning**: Python ecosystem utilizing [TensorFlow](https://www.tensorflow.org/), Keras, Librosa, and Spleeter for audio processing and chord prediction.
* **Cloud & Storage**: Google Cloud Platform (GCS) for storing audio files and model artifacts.

## Getting Started & Deployment

To deploy the project yourself, either locally or on Google Cloud, you will need to configure the environment. Scattered instructions on deploying with Minikube can be found at `team3/server/using-minikube.md`.

### 1. Configuration Files

You must create the following configuration files before building and running the project:

#### Google Cloud Credentials
Place your Google Cloud Service Account JSON in the following locations:
* `team3/server/validation_training_service/HarmonAI_storage.json`
* `team3/server/prediction_service/HarmonAI_storage.json`

```json
{
  "type": "service_account",
  "project_id": "",
  "private_key_id": "",
  "private_key": "",
  "client_email": "",
  "client_id": "",
  "auth_uri": "",
  "token_uri": "",
  "auth_provider_x509_cert_url": "",
  "client_x509_cert_url": "",
  "universe_domain": "googleapis.com"
}
```

#### Database Connection
Create an environment file at `team3/server/users_project/.env`:
```env
DB_CONNECTION_STRING=<MongoDB database URL w/ username + password>
```

#### Frontend Configuration
Create an environment file for production at `team3/client/harmonai_frontend/.env.production`:
```env
VITE_API_URL=<URL/address to where the site will be hosted>
```

For local development, create `team3/client/harmonai_frontend/.env`:
```env
VITE_GCS_BUCKET_NAME=
VITE_GCS_CLIENT_ID=
VITE_GCS_SCOPE=
VITE_API_URL=<URL/address to where the site will be hosted>
```
